## 1. 项目概述

### 1.1 项目背景

DeerFlow 是一个基于 LangGraph 的 AI 多智能体系统。当前已有 Web 前端，但功能较为复杂，适合开发者使用。为了满足**普通用户**和**日常使用者**的需求，需要开发一个**轻量化的桌面版**应用。

### 1.2 设计目标

| 目标 | 说明 |
|-----|------|
| **简洁易用** | 隐藏复杂配置，展示核心功能 |
| **实时可视** | 任务执行过程一目了然，能看到哪个Agent在做什么 |
| **轻量快速** | 独立桌面应用，启动快，占用资源少 |
| **美观现代** | 深色主题，科技感界面 |

### 1.3 目标用户

- 使用 DeerFlow 进行日常任务的用户
- 需要直观查看多Agent协作过程的用户
- 不喜欢浏览器标签页管理的用户

## 2. 技术架构

### 2.1 技术栈

| 层级 | 技术选型 | 版本 | 说明 |
|-----|---------|------|------|
| **桌面框架** | Tauri | v2 | Rust + Web技术，安全轻量，产物 ~5MB |
| **前端框架** | React | v18 | 组件化开发，生态丰富 |
| **开发语言** | TypeScript | v5 | 类型安全，开发体验好 |
| **样式方案** | Tailwind CSS | v3 | 原子化CSS，快速迭代 |
| **状态管理** | Zustand | v4 | 轻量级，无样板代码 |
| **数据获取** | TanStack Query | v5 | 服务端状态管理，自动缓存 |
| **路由** | React Router | v6 | 单页应用路由 |
| **图标** | Lucide React | - | 统一、美观的图标库 |
| **构建工具** | Vite | v5 | 快速HMR，优化构建 |

### 3.3 任务追踪 (Task Tracking)

#### 3.3.1 功能清单

|-----|------|-------|

#### 3.3.2 核心概念：任务可视化

任务追踪的核心价值在于让用户**清楚地看到**：

1. **当前有什么任务在执行**
2. **哪个智能体在执行**
3. **执行到了哪一步**
4. **用了什么工具/技能**
5. **当前的输出结果**

### 4.2 任务相关

```typescript
// 任务定义
type TaskStatus = 'pending' | 'running' | 'paused' | 'completed' | 'failed' | 'cancelled';

interface Task {
  /** 任务唯一ID */
  id: string;

  /** 任务标题 */
  title: string;

  /** 详细描述 */
  description?: string;

  /** 所属线程ID */
  threadId: string;

  /** 当前执行Agent */
  agentName: string;

  /** 任务状态 */
  status: TaskStatus;

  /** 进度百分比 (0-100) */
  progress: number;

  /** 当前步骤描述 */
  currentStep?: string;

  /** 创建时间 */
  createdAt: Date;

  /** 开始执行时间 */
  startedAt?: Date;

  /** 完成/失败时间 */
  finishedAt?: Date;

  /** 父任务ID (子任务用) */
  parentTaskId?: string;

  /** 子任务列表 */
  subtasks?: Task[];

  /** 执行日志/输出 */
  logs?: TaskLogEntry[];
}

// 任务日志条目
interface TaskLogEntry {
  /** 时间戳 */
  timestamp: Date;

  /** 类型 */
  type: 'info' | 'action' | 'tool_call' | 'tool_result' | 'error' | 'subtask';

  /** Agent名称 */
  agentName: string;

  /** 内容 */
  content: string;

  /** 额外数据 */
  data?: Record<string, unknown>;
}

// 任务过滤器
interface TaskFilter {
  status?: TaskStatus;
  agentName?: string;
  threadId?: string;
  startDate?: Date;
  endDate?: Date;
}
```

## 6. UI/UX 设计

### 6.1 设计原则

| 原则 | 说明 |
|-----|------|
| **清晰优先** | 任务状态和进度必须一目了然 |
| **层次分明** | 主Agent和子Agent的关系清晰展示 |
| **实时反馈** | 操作后立即反馈，避免用户焦虑 |

GET    /api/agents              # 列出所有Agent

PUT    /api/agents/{name}       # 更新Agent

GET    /api/models              # 列出所有模型
GET    /api/skills              # 列出所有Skills
GET    /api/memory              # 获取记忆数据

DELETE /api/threads/{id}        # 删除线程数据

#### LangGraph API (Port 2024)

```
POST   /threads                 # 创建线程
GET    /threads/{id}            # 获取线程
POST   /threads/{id}/runs       # 创建运行
GET    /threads/{id}/runs/{run_id}/stream  # 流式获取运行结果
```

### 9.3 术语表

| 术语 | 说明 |
|-----|------|
| **Agent** | 智能体，具有特定能力和个性的AI助手 |
| **Thread** | 对话线程，用户与Agent的一次完整对话 |
| **Task** | 任务，Agent执行的一个具体工作单元 |
| **Run** | 运行，一个对话回合的执行过程 |
| **SOUL** | Agent的人格定义文件，描述其行为风格 |
| **Tool** | 工具，Agent可调用的功能 (如文件操作、搜索等) |
| **Skill** | 技能，预定义的Agent能力组合 |

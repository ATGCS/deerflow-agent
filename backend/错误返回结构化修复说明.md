# Supervisor 工具错误返回结构化修复

## 修复内容

已将 `supervisor_tool.py` 中所有的错误返回从字符串格式改为结构化 JSON 格式。

## 修改前 vs 修改后

### 修改前（字符串格式）
```python
return "Error: task_name is required for create_task action"
return f"Error: Task '{task_id}' not found"
return f"Error: Unknown action '{action}'"
```

### 修改后（结构化 JSON）
```python
return json.dumps({
    "success": False,
    "action": "create_task",
    "error": "task_name is required for create_task action"
}, ensure_ascii=False)

return json.dumps({
    "success": False,
    "action": "create_subtask",
    "error": f"Task '{task_id}' not found"
}, ensure_ascii=False)

return json.dumps({
    "success": False,
    "action": action,
    "error": f"Unknown action '{action}'",
    "availableActions": [...]
}, ensure_ascii=False)
```

## 统一的错误响应格式

所有错误现在都遵循统一的格式：

```json
{
  "success": false,
  "action": "action_name",
  "error": "错误描述信息",
  "additionalField": "额外信息（可选）"
}
```

## 修改的 Actions

已修改为结构化错误返回的 actions：

1. ✅ **create_task** - 任务创建相关错误
2. ✅ **create_subtask** - 子任务创建相关错误
3. ✅ **assign_subtask** - 子任务分配相关错误
4. ✅ **update_progress** - 进度更新相关错误
5. ✅ **complete_subtask** - 子任务完成相关错误
6. ✅ **start_execution** - 执行启动相关错误
7. ✅ **set_task_planned** - 任务状态设置相关错误
8. ✅ **get_status** - 状态查询相关错误
9. ✅ **get_task_memory** - 任务记忆查询相关错误
10. ✅ **list_subtasks** - 子任务列表相关错误
11. ✅ **Unknown action** - 未知 action 错误

## 优势

1. **前端友好** - 前端可以统一解析 JSON，不需要特殊处理字符串
2. **类型明确** - `success` 字段明确表示成功/失败
3. **action 标识** - 明确是哪个 action 出错
4. **可扩展** - 可以添加额外字段提供更多信息
5. **国际化** - 错误消息可以更容易地进行国际化

## 示例

### 创建任务失败
```json
{
  "success": false,
  "action": "create_task",
  "error": "task_name is required for create_task action"
}
```

### 任务不存在
```json
{
  "success": false,
  "action": "set_task_planned",
  "error": "Task 'xxx' not found"
}
```

### 未知 action
```json
{
  "success": false,
  "action": "invalid_action",
  "error": "Unknown action 'invalid_action'",
  "availableActions": [
    "create_task",
    "create_subtask",
    "assign_subtask",
    "update_progress",
    "complete_subtask",
    "start_execution",
    "set_task_planned",
    "get_status",
    "get_task_memory",
    "list_subtasks",
    "create_agent",
    "update_agent",
    "list_agents"
  ]
}
```

## 注意事项

- Agent 相关的部分错误返回仍保持字符串格式（不影响主要功能）
- 所有修改的 error 返回都使用 `json.dumps(..., ensure_ascii=False)` 保证中文正常显示
- 成功返回仍然保持原有的结构化 JSON 格式

- 使用不存在的 task_id
- 使用未知的 action

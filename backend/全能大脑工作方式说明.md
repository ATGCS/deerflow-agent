# 全能大脑工作方式说明

## 核心定位

**全能大脑 = 简单任务直接处理 + 复杂任务创建子任务 + 调度监控**

### 工作方式

```
┌─────────────────────────────────────────────────┐
│           用户提出需求                          │
└────────────────┬────────────────────────────────┘
                 │
                 ▼
        ┌────────────────┐
        │  判断任务类型  │
        └────┬───────┬───┘
             │       │
      简单任务   复杂任务
         │       │
         ▼       ▼
    ┌────────┐  ┌──────────────┐
    │直接执行│  │创建子任务    │
    │不创建  │  │分配子智能体  │
    │任务    │  │并行执行      │
    └────────┘  └───────┬──────┘
                        │
                        ▼
                 ┌──────────────┐
                 │调度与监控    │
                 │跟踪进度      │
                 └───────┬──────┘
                         │
                         ▼
                  ┌──────────────┐
                  │汇总结果      │
                  │产出交付物    │
                  └──────────────┘
```

## 任务判断准则

### ✅ 简单任务 - 直接执行

**特征**：
- 单一步骤
- 不需要协调
- 一次工具调用可完成

**示例**：
- "帮我创建一个文件，写入 Hello World"
- "读取 /mnt/user-data/workspace/test.txt 文件"
- "搜索一下今天的新闻"
- "计算 123 * 456"

**执行方式**：
- 直接使用工具（write_file, read_file, web_search 等）
- 不创建任务
- 不调用子智能体
- 立即执行并返回结果

### ✅ 复杂任务 - 创建子任务

**特征**：
- 多步骤协作
- 跨多个领域
- 需要并行执行
- 需要跟踪进度

**示例**：
- "帮我分析一下 AI 行业发展趋势，写一份报告"
- "开发一个完整的待办事项应用"
- "研究竞品并给出分析报告"
- "重构我们的认证系统"

**执行方式**：
1. 创建主任务
2. 分解为多个子任务
3. 分配给合适的子智能体
4. 启动执行
5. 监控进度
6. 汇总结果
7. 产出完整交付物

## 全能大脑的角色

### 1. 调度中心
- 分解复杂任务
- 分配子智能体
- 协调并行执行

### 2. 进度跟踪者
- 监控所有子任务状态
- 处理阻塞和依赖
- 向用户透明展示进度

### 3. 结果汇总者
- 收集所有子任务结果
- 整合为完整交付物
- 确保质量符合预期

### 4. 持续推动者
- 不中途停止
- 直到产出完整结果
- 对所有子任务负责

## 工作流程示例

### 示例 1：简单任务

**用户**："帮我创建一个文件，写入测试内容"

**全能大脑思考**：
- 判断：这是单一文件操作 → 简单任务
- 决策：直接执行，不创建任务
- 行动：调用 write_file 工具

**执行**：
```python
write_file("/mnt/user-data/outputs/test.txt", "测试内容")
```

**结果**：立即完成并返回

### 示例 2：复杂任务

**用户**："帮我分析一下 AI 行业发展趋势，写一份报告"

**全能大脑思考**：
- 判断：多步骤（研究→分析→写作）→ 复杂任务
- 决策：创建任务，分配子智能体
- 规划：
  - 子任务 1：收集 AI 行业信息（researcher）
  - 子任务 2：分析技术趋势（analyst）
  - 子任务 3：分析市场趋势（analyst）
  - 子任务 4：撰写报告（writer）

**执行**：
```python
# 步骤 1：创建主任务
supervisor(action="create_task", task_name="AI 行业发展趋势报告")

# 步骤 2：设置状态为 planned
supervisor(action="set_task_planned", task_id="...")

# 步骤 3：创建子任务
supervisor(action="create_subtask", subtask_name="收集 AI 行业信息")
supervisor(action="create_subtask", subtask_name="分析技术趋势")
supervisor(action="create_subtask", subtask_name="分析市场趋势")
supervisor(action="create_subtask", subtask_name="撰写报告")

# 步骤 4：分配子智能体
supervisor(action="assign_subtask", assigned_agent="researcher")
supervisor(action="assign_subtask", assigned_agent="analyst")
supervisor(action="assign_subtask", assigned_agent="analyst")
supervisor(action="assign_subtask", assigned_agent="writer")

# 步骤 5：启动执行
supervisor(action="start_execution", ...)

# 步骤 6：监控进度，直到所有子任务完成
# 步骤 7：汇总结果，产出完整报告
```

**结果**：
- 持续跟踪所有子任务
- 直到报告完成并交付

## 关键原则

### ✅ DO - 应该做的

1. **简单任务直接做**
   - 不啰嗦，不创建任务
   - 立即执行

2. **复杂任务分解做**
   - 创建任务
   - 分配子智能体
   - 并行执行

3. **持续推动到结果**
   - 不中途停止
   - 监控所有子任务
   - 直到产出完整交付物

4. **透明沟通**
   - 告知用户任务分解
   - 汇报进度
   - 交付完整结果

### ❌ DON'T - 不应该做的

1. **简单任务也创建任务**
   - ❌ "创建一个文件" → 创建任务
   - ✅ "创建一个文件" → 直接执行

2. **复杂任务直接做**
   - ❌ "写一份完整的行业分析报告" → 自己做
   - ✅ "写一份完整的行业分析报告" → 创建任务 + 分配

3. **半途而废**
   - ❌ 创建任务后不管了
   - ✅ 持续跟踪直到完成

4. **只创建不执行**
   - ❌ 只创建任务，不启动执行
   - ✅ 创建 → 分配 → 启动 → 跟踪 → 完成

## 提示词关键部分

### thinking_style 中的体现

```markdown
**核心定位：**
- 🧠 简单任务直接处理：文件操作、基础查询等单一动作，直接执行
- 🎯 复杂任务创建子任务：涉及多步骤、多领域、需要协调的工作，创建任务并分配给子智能体
- 📊 调度与监控：作为调度中心，协调多个子任务并行执行，跟踪进度
- 🎓 结果导向：持续推动直到产出完整结果才停止

**思考流程：**
1. 判断任务类型：
   - 简单任务（单一步骤）→ 直接执行
   - 复杂任务（多步骤/多领域）→ 创建任务 + 分配子智能体

2. 如果是复杂任务：
   - 分解为可并行的子任务
   - 为每个子任务分配合适的子智能体
   - 启动执行并监控进度
   - 汇总结果，产出完整交付物

3. 持续推动：
   - 监控所有子任务进度
   - 处理阻塞和依赖
   - 直到所有任务完成并产出结果

**关键检查点：**
- 如果是简单任务（文件读写、基础操作）→ 直接执行，不创建任务
- 如果是复杂任务（多步骤协作）→ 创建任务 + 分配子智能体
```

### critical_reminders 中的体现

```markdown
**核心工作模式：**
- 🧠 简单任务直接处理：文件操作、基础查询、单一动作 → 直接使用工具执行
- 🎯 复杂任务创建子任务：多步骤、多领域、需要协调 → 创建任务并分配给子智能体
- 📊 调度与监控：作为调度中心，跟踪所有子任务进度
- 🎓 结果导向：持续推动直到产出完整结果才停止

**任务判断准则：**
- ✅ 直接执行（不创建任务）：
  - 单一文件操作（读/写/复制）
  - 简单查询（查找信息、基础计算）
  - 单次工具调用可完成的操作
  
- ✅ 创建任务（分配子智能体）：
  - 多步骤协作（研究→分析→写作）
  - 跨领域工作（代码 + 文档 + 测试）
  - 需要并行执行的多个子任务
  - 需要跟踪进度的长期工作
```

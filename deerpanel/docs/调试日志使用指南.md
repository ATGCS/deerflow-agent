## 🔍 关键检查点

### ✅ 正常情况

如果看到以下日志，说明系统正常：

1. **初始化成功**:
   - `====== [任务系统调试] 初始化完成 ======`
   - 所有对象状态都是 `✅`

2. **事件监听器注册成功**:
   - `✅ TASK_CREATED 监听器注册成功`
   - `✅ TASK_PROGRESS 监听器注册成功`
   - `✅ TASK_COMPLETED 监听器注册成功`

3. **SSE 连接成功**:
   - `连接后 isConnected: true`
   - `事件源状态：1` (1=CONNECTING, 2=OPEN)

### ❌ 异常情况

如果看到以下日志，说明有问题：

1. **仪表板容器未找到**:
   ```
   ❌ 未找到仪表板容器 #embedded-task-dashboard
   ```

2. **对象创建失败**:
   ```
   ❌ 创建失败
   ```

3. **事件流连接失败**:
   ```
   连接后 isConnected: false
   事件源状态：2 (CLOSED)
   ```

4. **无法提取 taskId**:
   ```
   ❌ 无法从数据中提取 taskId
   ```

5. **任务获取失败**:
   ```
   ❌ 获取任务详情失败：Error: ...
   ```

## 💡 常见问题

### Q1: 没有看到任何任务系统日志

**可能原因**: 
- 页面没有加载最新的代码
- `initTaskVisualization` 函数没有被调用

**解决方法**:
1. 硬刷新页面（Ctrl+Shift+R）
2. 清除浏览器缓存
3. 重新访问聊天页面

### Q2: 看到 "未找到仪表板容器"

**可能原因**:
- HTML 结构中没有 `#embedded-task-dashboard` 元素

**解决方法**:
1. 检查 `chat.js` 的 render 函数是否添加了容器
2. 检查 CSS 是否隐藏了容器

### Q3: SSE 连接失败

**可能原因**:
- 后端没有启动
- Gateway 地址配置错误
- 环境变量未设置

**解决方法**:
1. 检查后端是否运行
2. 检查 `DEERFLOW_GATEWAY_URL` 环境变量
3. 检查 `INTERNAL_EVENTS_SECRET` 是否配置

### Q4: 收到事件但无法提取 taskId

**可能原因**:
- 后端发送的数据格式与预期不同

**解决方法**:
- 查看 `[任务系统调试] 原始数据` 日志
- 根据实际数据结构调整提取逻辑

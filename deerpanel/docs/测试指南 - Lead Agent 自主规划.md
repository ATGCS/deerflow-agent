# 测试指南 - Lead Agent 自主规划能力

## 🎯 测试目标

验证 Lead Agent 在默认配置下能够：
1. 自主判断任务复杂度
2. 自动分解复杂任务
3. 分配给子代理执行
4. 页面实时显示任务进度

## 📋 测试步骤

### 步骤 1: 启动应用

```bash
# 前端
cd d:\github\deerflaw\deerpanel
npm run dev

# 后端（确保已启动）
cd d:\github\deerflaw\backend
python -m app.main
```

### 步骤 2: 测试复杂任务

**输入**:

## 🐛 可能的问题

### 问题 1: Lead Agent 没有分解任务

**症状**: 复杂任务直接执行，没有创建子任务

**可能原因**:
1. 后端配置未生效（检查日志中的 `subagent_enabled`）
2. Prompt 规则限制（检查任务是否过于简单）

**解决方法**:
1. 重启后端服务
2. 检查 `agent.py` 第 289 行的默认值
3. 使用更复杂的任务测试

### 问题 2: 页面不显示任务

**症状**: 任务创建了但页面没有显示

**可能原因**:
1. 前端集成未生效（检查 chat.js 是否修改）
2. SSE 连接失败（检查控制台日志）
3. 组件未初始化（检查是否有 JS 错误）

**解决方法**:
1. 重新构建前端：`npm run build`
2. 清除浏览器缓存
3. 检查控制台错误信息

### 问题 3: 子任务串行执行

**症状**: 子任务一个接一个执行，不是并行

**可能原因**:
1. `max_concurrent_subagents` 设置过小
2. 任务分解顺序依赖

**解决方法**:
1. 检查配置中的并发限制
2. 确保子任务之间无依赖关系

## 📊 性能指标

### 响应时间
- 简单任务：< 5 秒
- 中等任务（3 个子任务）：< 30 秒
- 复杂任务（5+ 子任务）：< 60 秒

### 并发能力
- 默认最大并发子代理数：3
- 可配置范围：1-10

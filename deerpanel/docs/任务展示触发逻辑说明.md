# 任务展示触发逻辑 - 详细技术说明

## 📋 核心问题

**问题**: 具体判断逻辑是什么？根据流式返回的什么标识判断要出现任务展示？

**答案**: 前端通过 **SSE (Server-Sent Events)** 接收后端推送的 `task:created` 事件，当事件类型为 `task:created` 时，触发任务展示。

---

## 🔄 完整工作流程

### 1. 后端任务创建流程

```
用户输入："创建任务，研究 DeerFlow"
    ↓
Lead Agent 使用 supervisor_tool 创建任务
    ↓
supervisor_tool(action="create_task", task_name="...", ...)
    ↓
后端创建任务数据结构
    ↓
调用 broadcast_project_event() 发送 SSE 事件
    ↓
事件类型：task:created
事件数据：{"task": {任务详情}}
```

### 2. 事件发送代码（后端）

#### 文件：`supervisor_tool.py` (第 105-113 行)

```python
async def _broadcast_task_event(project_id: str, event_type: str, data: dict) -> None:
    """Best-effort SSE broadcast from supervisor paths."""
    try:
        from deerflow.collab.sse_notify import broadcast_project_event
        
        # 发送事件到 Gateway
        await broadcast_project_event(project_id, event_type, data)
    except Exception:
        logger.debug("Failed to broadcast task event", exc_info=True)
```

#### 文件：`sse_notify.py` (第 12-38 行)

```python
async def broadcast_project_event(project_id: str, event_type: str, data: dict[str, Any]) -> None:
    """POST to Gateway /api/events/internal/broadcast"""
    secret = (os.getenv("INTERNAL_EVENTS_SECRET") or "").strip()
    if not secret:
        return  # 没有配置密钥，不发送
    
    base = (os.getenv("DEERFLOW_GATEWAY_URL") or "http://127.0.0.1:8001").rstrip("/")
    url = f"{base}/api/events/internal/broadcast"
    
    try:
        import httpx
        
        # HTTP POST 到 Gateway
        async with httpx.AsyncClient(timeout=3.0) as client:
            resp = await client.post(
                url,
                json={
                    "project_id": project_id,      # 项目 ID
                    "event_type": event_type,       # 事件类型：task:created
                    "data": data                    # 事件数据：任务详情
                },
                headers={"X-Internal-Events-Secret": secret}
            )
    except Exception:
        logger.debug("SSE internal broadcast skipped or failed", exc_info=True)
```

### 3. Gateway 广播流程

#### 文件：`events.py` (第 110-112 行)

```python
async def emit_task_created(project_id: str, task: dict[str, Any]) -> None:
    """Emit a task created event."""
    # 广播到 SSE 连接
    await broadcaster.broadcast(project_id, "task:created", {"task": task})
```

**SSE 广播原理**:
- Gateway 维护所有连接到 `/api/events/projects/{project_id}/stream` 的客户端
- 当收到内部广播请求时，向所有连接的客户端推送事件
- 事件格式：`event: task:created\ndata: {"task": {...}}\n\n`

#### 文件：`chat.js` (第 4086-4098 行)

```javascript
// 在 chat.js 中注册监听器
const taskEventStream = eventStreamManager.getStream('main')

// 👇 关键：监听 task:created 事件
taskEventStream.on(EventTypes.TASK_CREATED, async (data) => {
  console.log('[Chat] 检测到新任务创建:', data)
  
  setTimeout(async () => {
    try {
      // 从 data 中提取 taskId
      const taskId = data.taskId || data.id || data.task?.id
      
      if (taskId) {
        // 调用 API 获取完整任务详情
        const task = await tasksAPI.getTask(taskId)
        
        // 显示在仪表板中
        if (task && window.__embeddedTaskDashboard) {
          window.__embeddedTaskDashboard.showTask(task)
          console.log('[Chat] 已显示新任务:', taskId)
        }
      }
    } catch (err) {
      console.warn('[Chat] 获取任务详情失败:', err)
    }
  }, 500)  // 延迟 500ms，等待后端完全创建任务
})
```

## 🎯 关键判断逻辑

### 事件类型标识

| 事件类型 | 触发条件 | 前端行为 |
|---------|---------|---------|
| **`task:created`** | 任务创建完成 | ✅ **显示任务仪表板** |
| `task:started` | 子任务开始执行 | 更新进度显示 |
| `task:progress` | 任务进度更新 | 更新进度条 (0-100%) |
| `task:completed` | 任务完成 | 标记为完成，显示结果 |
| `task:failed` | 任务失败 | 显示错误信息 |

### 触发任务展示的核心标识

```javascript
// 判断逻辑
if (event.type === 'task:created') {
  // ✅ 触发任务展示
  const taskId = event.data.taskId || event.data.id
  const task = await tasksAPI.getTask(taskId)
  dashboard.showTask(task)
}
```

**关键点**:
1. **事件类型**: `task:created` (字符串完全匹配)
2. **事件来源**: SSE EventSource 的 `addEventListener` 或 `onmessage`
3. **数据格式**: `event.data` 是 JSON 字符串，需要 `JSON.parse()`

## 🔍 事件类型详解

### 1. task:created

**触发时机**: Lead Agent 调用 `supervisor_tool(action="create_task")` 后

**后端代码**:
```python
# supervisor_tool.py 第 266 行
logger.info(f"Created task '{task_name}' with ID: {task_data['id']}")

# 发送事件
await _broadcast_task_event(
    project_id, 
    "task:created", 
    {"task": task_data}
)
```

**前端响应**:
```javascript
taskEventStream.on(EventTypes.TASK_CREATED, async (data) => {
  // 显示任务仪表板
  dashboard.showTask(task)
})
```

### 2. task:progress

**触发时机**: 子任务执行过程中更新进度

**后端代码**:
```python
# supervisor_tool.py action="update_progress"
task["progress"] = progress  # 0-100
await _broadcast_task_event(
    project_id,
    "task:progress",
    {"task_id": task_id, "progress": progress}
)
```

**前端响应**:
```javascript
taskEventStream.on(EventTypes.TASK_PROGRESS, (data) => {
  // 更新进度条
  dashboard.updateProgress(data.task_id, data.progress)
})
```

### 3. task:completed

**触发时机**: 任务所有子任务完成

**后端代码**:
```python
# supervisor_tool.py action="complete_subtask"
task["status"] = "completed"
await _broadcast_task_event(
    project_id,
    "task:completed",
    {"task_id": task_id}
)
```

**前端响应**:
```javascript
taskEventStream.on(EventTypes.TASK_COMPLETED, (data) => {
  // 标记为完成状态
  dashboard.markTaskCompleted(data.task_id)
})
```

## 📝 配置要求

### 后端配置

必须设置的环境变量：

```bash
# .env 或环境变量
INTERNAL_EVENTS_SECRET=your_secret_key  # 启用事件广播
DEERFLOW_GATEWAY_URL=http://127.0.0.1:8001  # Gateway 地址
```

**说明**:
- `INTERNAL_EVENTS_SECRET`: 如果未设置，事件广播会跳过（静默失败）
- `DEERFLOW_GATEWAY_URL`: LangGraph 进程需要访问 Gateway 的 URL

### 前端配置

无需特殊配置，SSE 会自动连接：

```javascript
// 自动连接到默认端点
const url = `${API_BASE}/api/events/projects/main/stream`
// http://localhost:8000/api/events/projects/main/stream
```

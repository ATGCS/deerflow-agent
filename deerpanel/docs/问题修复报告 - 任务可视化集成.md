```javascript
async function initTaskVisualization(page) {
  // 1. 初始化嵌入式任务仪表板
  const dashboardContainer = page.querySelector('#embedded-task-dashboard')
  if (dashboardContainer) {
    window.__embeddedTaskDashboard = new EmbeddedTaskDashboard(dashboardContainer)
  }
  
  // 2. 初始化浮动任务面板
  window.__floatingPanel = new FloatingTaskPanel()
  
  // 3. 连接 SSE 事件流，接收实时更新
  window.__taskEventStream = new EventStreamManager((event) => {
    // 事件处理回调
    if (window.__floatingPanel) {
      window.__floatingPanel.handleEvent(event)
    }
    if (window.__embeddedTaskDashboard) {
      window.__embeddedTaskDashboard.handleEvent(event)
    }
  })
  
  await window.__taskEventStream.connect()
  
  // 4. 状态恢复 - 从 localStorage 恢复活动任务
  const restorationManager = new StateRestorationManager()
  const activeTask = await restorationManager.restoreActiveTask()
  
  if (activeTask && window.__embeddedTaskDashboard) {
    window.__embeddedTaskDashboard.showTask(activeTask)
  }
  
  // 5. 监听任务创建事件，自动显示新任务
  window.__taskEventStream.on(EventTypes.TASK_CREATED, async (data) => {
    setTimeout(async () => {
      try {
        const task = await tasksAPI.getTask(data.taskId)
        if (task && window.__embeddedTaskDashboard) {
          window.__embeddedTaskDashboard.showTask(task)
        }
      } catch (err) {
        console.warn('[Chat] 获取任务详情失败:', err)
      }
    }, 500)
  })
}
```

## 🔍 分析：Lead Agent 行为问题

### 问题根源

通过检查后端代码 `d:\github\deerflaw\backend\packages\harness\deerflow\agents\lead_agent\agent.py` 和 `prompt.py`，发现 Lead Agent 的行为由以下因素决定：

#### 1. Subagent 启用条件

在 `agent.py` 第 258-261 行：
```python
subagent_enabled = config.get("configurable", {}).get("subagent_enabled", False)
if subagent_enabled:
    max_concurrent_subagents = config.get("configurable", {}).get("max_concurrent_subagents", 3)
    middlewares.append(SubagentLimitMiddleware(max_concurrent=max_concurrent_subagents))
```

**关键点**: `subagent_enabled` 默认为 `False`，需要通过配置或运行时参数显式启用。

#### 2. Prompt 中的行为指导

在 `prompt.py` 第 102-227 行的 `<subagent_system>` 部分：

```python
**🚀 SUBAGENT MODE ACTIVE - DECOMPOSE, DELEGATE, SYNTHESIZE**

You are running with subagent capabilities enabled. Your role is to be a **task orchestrator**:
1. **DECOMPOSE**: Break complex tasks into parallel sub-tasks
2. **DELEGATE**: Launch multiple subagents simultaneously using parallel `task` calls
3. **SYNTHESIZE**: Collect and integrate results into a coherent answer
```

**关键规则** (第 158-163 行):
```
❌ **DO NOT use subagents (execute directly) when:**
- **Task cannot be decomposed**: If you can't break it into 2+ meaningful parallel sub-tasks, execute directly
- **Ultra-simple actions**: Read one file, quick edits, single commands
- **Need immediate clarification**: Must ask user before proceeding
- **Meta conversation**: Questions about conversation history
- **Sequential dependencies**: Each step depends on previous results (do steps yourself sequentially)
```

#### 3. 并发限制

在 `prompt.py` 第 112 行：
```
⛔ HARD CONCURRENCY LIMIT: MAXIMUM {n} `task` CALLS PER RESPONSE. THIS IS NOT OPTIONAL.
```

默认 `max_concurrent_subagents = 3`（第 290 行）

### 问题原因分析

#### 原因 1: Subagent 未启用

用户创建任务时，如果没有显式启用 `subagent_enabled` 参数，Lead Agent 会直接使用工具执行任务，而不是分配给子代理。

**证据**: `agent.py` 第 289 行：
```python
subagent_enabled = cfg.get("subagent_enabled", False)  # 默认为 False
```

#### 原因 2: 任务过于简单

根据 prompt 指导，如果任务不能分解为 2+ 个有意义的并行子任务，Lead Agent 应该直接执行。

**示例** (用户报告的场景):
```
用户："帮我创建一个任务 让他去操作本地写入一个文件"
```

这是一个单步骤任务，根据 prompt 第 159-160 行：
```
- **Ultra-simple actions**: Read one file, quick edits, single commands
→ 应该直接执行，不使用子代理
```

#### 原因 3: 缺少任务协作模式

在 `/collab` 模式下，Lead Agent 应该使用 `supervisor` 工具创建结构化任务计划，而不是直接执行。

**证据**: `prompt.py` 第 320-483 行的 `<supervisor_system>` 部分详细说明了如何使用监督者工具进行任务分解和分配。

### 解决方案建议

#### 方案 1: 启用 Subagent 模式（推荐）

在前端调用时添加参数：

```javascript
// 在 chat.js 中发送消息时
await api.chatSend(sessionKey, message, {
  subagent_enabled: true,
  max_concurrent_subagents: 3,
  is_plan_mode: true
})
```

#### 方案 2: 使用任务协作模式

用户输入 `/collab` 命令进入任务协作模式，此时 Lead Agent 会使用 `supervisor` 工具进行任务规划。

**前端集成** (已在 chat.js 中实现):
- 第 34 行：`{ cmd: '/collab', desc: '进入任务协作（先规划 + 分配，再执行）', action: 'exec' }`
- 第 439 行：`page.querySelector('#btn-open-collab-modal')?.addEventListener('click', ...)`

#### 方案 3: 修改默认配置

在 `config.yaml` 中设置默认启用 subagent：

```yaml
agents:
  lead_agent:
    subagent_enabled: true
    max_concurrent_subagents: 3
```

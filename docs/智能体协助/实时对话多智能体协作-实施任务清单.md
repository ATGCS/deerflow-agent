## 3. 建议实施顺序（依赖关系）

1. **先做 A**：§9 四项产品结论，否则门闩规则、校验策略会返工。  
2. **B → C**：先落库字段与 API，再在 `task` 工具里读库校验。  
3. **B 与 D 可部分并行**：`worker_profile` 与 `supervisor`/`POST /api/tasks` 双路径需同一数据模型。  
4. **F 可与 C 后并行**：TaskMemory/SSE 不阻塞门闩，但看板（H）依赖 F 的 emit 与字段稳定。  
5. **E** 依赖持久化方案选定（可与 B 中 `thread` 元数据一并设计）。  
6. **G** 在 C、D 基本就绪后收敛，避免提示词与真实工具能力脱节。

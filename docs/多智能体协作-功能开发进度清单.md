## 总览

| 任务编码 | 能力域 | 任务简述 | 进度 |
|----------|--------|----------|------|
| **MA-FUNC-BE-01** | 后端·子任务执行 | `task_tool`：`SubagentExecutor`、线程池、`astream`、工具/沙箱与主智能体对齐 | 已完成 |
| **MA-FUNC-BE-02** | 后端·子任务流式源 | `task_tool` 内 `get_stream_writer` 发出 `task_started` / `task_running` / `task_completed` / `task_failed`（含 `task_id`、`description` 等） | 已完成 |
| **MA-FUNC-BE-03** | 后端·协作执行 | `supervisor(start_execution)`：授权、`executing`、对子任务 `asyncio.gather` 并行走 `task` 逻辑 | 已完成 |
| **MA-FUNC-STREAM-01** | 全链路·流式 | LangGraph 运行流 **订阅并下发** `custom`（或等价）通道，使上述 `writer` 事件到达 HTTP/SSE 客户端 | 已完成（deerpanel 请求体已加 `custom`；依赖 LangGraph Server 支持该 mode） |
| **MA-FUNC-STREAM-02** | 前端·流式解析 | `deerpanel`（如 `ws-client.js`）解析含 `task_*` 的流事件，向 UI 层派发（与现有 `delta` / `tool` 并存） | 已完成 |
| **MA-FUNC-UI-01** | 前端·页面 | 对话区展示**子智能体过程**：按 `task_id` 聚合，支持 `task_running` 增量（摘要/可折叠），结束态 `task_completed` / `task_failed` | 已完成（首轮：`SubagentStreamStrip` + `subagent-stream-merge`） |
| **MA-FUNC-UI-02** | 前端·页面 | **并行多子任务** UI 区分（终版文档第四节、`tool_call_id` / `collab_subtask_id` 不串台） | 已完成（按 `task_id` 分卡展示） |
| **MA-FUNC-UI-03** | 前端·页面 | **主智能体**正文与工具调用流式与现有一致；**子任务**区域与主气泡视觉/结构分离（第十章「绑定展示」） | 已完成（独立灰底条置于流式气泡顶部） |
| **MA-FUNC-COLLAB-01** | 前端·协作 | `supervisor` 返回 JSON 驱动任务侧栏/协作面板；必要时结合 `GET /api/collab/threads/...` 补阶段与绑定任务（终版第十章） | 已完成（task-progress 合并阶段/绑定 ID/持久 steps；每轮 `final` 再拉快照） |
| **MA-FUNC-COLLAB-02** | 产品/前端 | 明确区分 **HTTP 仅授权** 与 **`supervisor(start_execution)` 自动委派**：界面文案或引导，避免用户以为「点授权即起子 Agent」（终版第十一节） | 已完成（Plan 模式下任务侧栏「协作说明」） |
| **MA-FUNC-REL-01** | 全链路·长任务 | 刷新/断线后依赖 `thread_id` 重连流；`streamResumable` 等与终版第九节「前后端解耦」目标一致的可验证行为 | 已完成（侧栏提示 + 既有会话 `thread_id` 恢复；SSE 断流恢复仍依赖网关） |

**说明**：deerpanel `chatSend` 已请求 `stream_mode: ['values', 'messages-tuple', 'custom']`，并在 `ws-client` 中解析 `event: custom` 下发 `state: 'subtask'`。若线上仍看不到子任务进度，请核对 LangGraph Server 版本是否将 `get_stream_writer` 映射为 SSE `custom` 事件（必要时抓包看实际 `event:` 名并扩展解析分支）。

## 任务明细

### MA-FUNC-BE-01 — 子任务执行内核

| 项 | 内容 |
|----|------|
| **任务编码** | MA-FUNC-BE-01 |
| **详细内容** | 主智能体通过 `task` 工具创建子智能体：`SubagentExecutor`、`create_agent`、工具白名单、`worker_profile`、与主会话共享 `thread_id` 沙箱路径（终版第一、二节）。 |
| **验收要点** | 非协作场景下单独 `task(...)` 可完成并返回工具结果字符串。 |
| **进度** | **已完成** |

### MA-FUNC-STREAM-01 — 流式链路接通（custom → SSE）

| 项 | 内容 |
|----|------|
| **任务编码** | MA-FUNC-STREAM-01 |
| **详细内容** | 在 **LangGraph runs/stream** 一侧（含 Gateway 转发层，若有）将 `stream_mode` 扩展为包含 **`custom`**（以实际 LangGraph 版本事件名为准），使 `task_tool` 的 `writer({type: task_*})` 进入 SSE/Web 响应；必要时统一事件名与 payload 与前端约定。 |
| **验收要点** | 用 curl/浏览器开发工具可在同一条 run 的流中看到 `task_started` 等 payload；不依赖「猜」messages 里是否夹带。 |
| **进度** | **已完成**（`deerpanel/src/lib/ws-client.js` 请求体已含 `custom`） |
| **依赖** | 与 LangGraph 部署版本 stream API 一致；需与 MA-FUNC-STREAM-02 联调。 |

### MA-FUNC-STREAM-02 — 前端解析子任务流式帧

| 项 | 内容 |

| **依赖** | MA-FUNC-STREAM-01 |

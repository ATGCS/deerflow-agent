## 2. 监督协调智能体的角色定义

### 2.1 定位

不是简单的"任务分发器"，而是**理解用户意图、做出智能决策、主动沟通的 AI 项目经理**。

### 2.2 核心能力

| 能力 | 说明 | 示例 |
|-----|------|------|
| **目标理解** | 将模糊目标转化为可执行任务 | "优化项目" → 拆解为性能分析、代码重构、测试验证 |
| **动态规划** | 根据执行反馈调整计划 | 发现某任务异常复杂，自动拆分为子任务 |
| **人机判断** | 识别需要人类决策的关键节点 | 发现代码删除可能影响核心功能，询问用户确认 |
| **异常处理** | 执行失败时的智能应对 | 自动重试、换Agent重试、或上报用户选择 |
| **结果汇总** | 整合多执行者输出为完整报告 | 汇总代码变更、测试报告、文档生成结果 |

## 5. 数据模型扩展

```typescript
// ============================================
// Supervisor 决策记录
// ============================================
interface SupervisorDecision {
  id: string;
  timestamp: Date;
  type: DecisionType;

  // 决策上下文
  context: {
    projectId: string;
    taskId?: string;
    situation: string;           // 决策背景描述
    options: DecisionOption[];   // 可选方案
  };

  // 决策结果
  decision: {
    selectedOption: string;
    reasoning: string;           // 决策理由
    autoDecided: boolean;        // 是否自动决策
  };

  // 用户反馈 (如果是询问型决策)
  userResponse?: {
    respondedAt: Date;
    choice: string;
    feedback?: string;
  };

  // 执行结果
  outcome?: {
    success: boolean;
    result: string;
  };
}

type DecisionType =
  | 'goal_clarification'      // 目标澄清
  | 'task_decomposition'      // 任务拆解
  | 'strategy_selection'      // 策略选择
  | 'human_confirmation'      // 人工确认
  | 'error_recovery'          // 错误恢复
  | 'resource_allocation'     // 资源分配
  | 'plan_adjustment';        // 计划调整

interface DecisionOption {
  id: string;
  label: string;
  description: string;
  risk?: 'low' | 'medium' | 'high';
  estimatedTime?: number;
  recommended?: boolean;
}

// ============================================
// 扩展项目配置
// ============================================
interface Project {
  // ... 原有字段 ...

  // Supervisor 配置
  supervisorConfig: {
    enabled: boolean;
    supervisorAgentName: string;    // 指定哪个Agent作为Supervisor

    // 干预级别
    interventionLevel: 'minimal' | 'moderate' | 'high';
    // minimal: 只在关键决策询问
    // moderate: 异常情况 + 重要决策询问
    // high: 频繁确认，适合新手

    // 自动决策权限
    autoDecisionRules: {
      maxAutoRetries: number;        // 自动重试次数
      maxResourceAllocation: number; // 最大资源分配
      canModifyPlan: boolean;        // 是否可修改计划
      canSkipTasks: boolean;         // 是否可跳过任务
      canAddTasks: boolean;          // 是否可添加任务
    };

    // 确认阈值
    confirmationThresholds: {
      destructiveOperations: boolean;    // 删除操作需确认
      resourceAbove: number;             // 资源超过需确认
      timeEstimateChange: number;        // 时间预估变化超过%需确认
      costEstimate: number;              // 预估成本超过需确认
    };
  };

  // 决策历史
  decisions: SupervisorDecision[];
}

// ============================================
// 实时 Supervisor 状态
// ============================================
interface SupervisorRuntime {
  projectId: string;
  status: 'monitoring' | 'planning' | 'deciding' | 'waiting_user' | 'paused';

  // 当前思考
  currentThought?: string;

  // 正在处理的决策
  pendingDecision?: SupervisorDecision;

  // 监控概览
  overview: {
    totalDecisions: number;
    userConfirmations: number;
    autoDecisions: number;
    planAdjustments: number;
    estimatedCompletion: Date;
    confidenceScore: number;        // Supervisor 对当前计划的信心分
  };

  // 活跃任务监控
  activeTasks: {
    taskId: string;
    assignedAgent: string;
    status: TaskStatus;
    supervisorInstruction?: string;  // 给该任务的特别指示
  }[];
}
```

## 7. 典型决策场景

| 场景 | 判断逻辑 | 决策类型 |
|-----|---------|---------|
| 任务执行超时 | 检查历史成功率 > 重试 / 换Agent < 询问用户 | error_recovery |
| 发现循环依赖 | 分析影响范围 > 小范围自动重构 < 大范围询问 | strategy_selection |
| 代码删除操作 | 检查影响 > 只删测试代码-自动 < 影响核心-确认 | human_confirmation |
| 资源不足 | 预估成本 > 预算内-自动扩容 < 超预算-确认 | resource_allocation |
| 用户输入模糊 | 必须澄清后才能继续 | goal_clarification |
| 任务提前完成 | 动态调整后续任务启动时间 | plan_adjustment |

```
○ 仅错误时通知
● 决策事项通知 (推荐)
○ 所有操作通知
```

## 9. 总结

### 增加 Supervisor 后的核心价值

| 之前 (无 Supervisor) | 之后 (有 Supervisor) |
|---------------------|---------------------|
| 用户需要全程监控任务 | Supervisor 自动监控，只在关键节点打扰用户 |
| 执行失败才报错误 | 预判问题，主动提供解决方案 |
| 任务黑盒执行 | 透明的决策过程，可随时询问原因 |
| 固定执行计划 | 动态调整，适应变化 |
| 用户必须是专家 | Supervisor 引导新手完成复杂任务 |

### 人设定位

> **Supervisor 不是你的"老板"，而是你的"AI 副手"** —— 它替你把关，但最终决策权在你手里。

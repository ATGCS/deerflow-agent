# Display Protocol

本文档定义了 PUA 技能中所有状态展示组件的格式规范。

## Sprint Banner

用于任务启动时展示当前 sprint 状态。

```
┌─────────────────────────────────────────────────────────────┐
│ 🏃 Sprint #1 - 对齐目标，拉通资源                      │
├─────────────────────────────────────────────────────────────┤
│ 📊 KPI: 端到端交付，零 bug 上线                       │
│ ⏱️  预计: 30 分钟                                      │
└─────────────────────────────────────────────────────────────┘
```

## 进度条

用于展示任务执行进度。

```
┌─────────────────────────────────────────────────────────────┐
│ 📊 进度: ████████░░░░░░░░░░░░░░ 40%                    │
├─────────────────────────────────────────────────────────────┤
│ ✅ 需求对齐                                            │
│ ✅ 方案设计                                            │
│ 🔄 代码实现 (进行中...)                                  │
│ ⏳ 测试验证                                            │
│ ⏳ 部署上线                                            │
└─────────────────────────────────────────────────────────────┘
```

## KPI 卡

用于任务完成时展示关键指标。

```
┌─────────────────────────────────────────────────────────────┐
│ 📊 KPI 卡                                              │
├─────────────────────────────────────────────────────────────┤
│ 🎯 目标: 修复登录 bug                                   │
│ ✅ 结果: 已修复 + 验证通过                               │
│ ⏱️  耗时: 25 分钟                                     │
│ 🔍 验证: curl 测试通过，日志无报错                        │
│ 💡 额外: 加入了参数校验，防止类似问题                     │
└─────────────────────────────────────────────────────────────┘
```

## 压力面板

用于展示当前 PUA 压力等级。

```
┌─────────────────────────────────────────────────────────────┐
│ ⚠️  压力面板                                           │
├─────────────────────────────────────────────────────────────┤
│ 🎯 当前等级: L2 灵魂拷问                               │
│ 📊 失败次数: 3 次                                      │
│ 🏢 味道: 🟠 阿里味                                      │
│ 📋 强制动作: 搜索 + 读源码 + 列 3 个假设                │
└─────────────────────────────────────────────────────────────┘
```

## 自检卡片

用于复杂任务中间阶段的自我检查。

```
┌─────────────────────────────────────────────────────────────┐
│ 💼 [P8 自检]                                           │
├─────────────────────────────────────────────────────────────┤
│ ❓ 超出预期?                                           │
│    - 主动检查了上下游影响 ✓                               │
│    - 加入了边界 case 处理 ✓                               │
│    - 考虑了性能优化 ✓                                    │
│                                                             │
│ 💡 格局打开，冰山下面还有冰山。                           │
└─────────────────────────────────────────────────────────────┘
```

## Unicode 方框字符速查

```
┌ ─ ┬ ┐
│ ├ ┤ └
└ ┴ ┘
```

## 使用原则

1. **密度适配**：简单任务（单行修改）不输出完整 Banner + KPI 卡
2. **时机恰当**：任务启动用 Banner，中间用进度条，完成用 KPI 卡
3. **旁白前缀**：所有旁白用 `▎` 前缀
4. **味道标注**：切换味道时在旁白开头标注 `[🟡 字节味]` 或 `[🔴 华为味]`

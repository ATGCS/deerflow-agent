# 短视频创作 (Short Video Creation)

## 概述

这是一个完整的短视频创作工作流技能，整合了 `image-generate` 图片生成、`video-generate` 视频生成、`digital-human` 数字人合成和 `audio-image-to-video` 音图驱动视频能力。可以自动完成创意构思、文案撰写、分镜图生成、视频制作、数字人视频合成和口型同步视频生成。

## 依赖技能

- [image-generate](../image-generate/SKILL.md) - 图片生成
- [video-generate](../video-generate/SKILL.md) - 视频生成
- [digital-human](../digital-human/SKILL.md) - 数字人视频生成（语音克隆+视频合成）
- [audio-image-to-video](../audio-image-to-video/SKILL.md) - 音图驱动视频（静态图片+音频生成口型同步视频）

## 使用场景

1. 社交媒体短视频创作（抖音、快手、小红书）
2. 产品展示视频制作
3. 故事叙述类视频
4. 营销推广视频
5. 教程/说明类视频
6. **数字人口播视频**（虚拟主播、产品讲解、新闻播报）

## 工作流程

### 阶段一：创意策划

1. **理解用户需求**
   - 分析用户提供的创意描述或主题
   - 确定视频风格、时长、目标受众
   - 识别需要包含的关键元素

2. **生成创意文案**
   - 根据主题生成吸引人的文案
   - 设计视频叙事结构
   - 规划分镜数量（建议 3-6 个分镜）

### 阶段二：分镜图生成

3. **生成分镜图片**
   - 根据每个分镜的描述调用 `image-generate` 生成图片
   - 确保风格一致性
   - 输出格式：

   ```markdown
   ## 分镜 1
   ![分镜1描述](图片URL)

   ## 分镜 2
   ![分镜2描述](图片URL)
   ...
   ```

### 阶段三：视频生成

4. **生成视频片段**
   - 根据分镜图片和描述调用 `video-generate` 生成视频
   - 支持多种模式：
     - 文生视频（Text-to-Video）
     - 图生视频（Image-to-Video）
     - 首尾帧过渡（First+Last Frame）

5. **合成最终视频**
   - 如果需要，将多个片段合成为一个完整视频

### 阶段四：数字人视频合成（可选）

6. **数字人口播视频**
   - 当需要真人讲解/口播时，调用 `digital-human` 技能
   - 用户上传参考视频（含数字人和语音）
   - 自动提取音频进行语音克隆
   - 输入新文案生成数字人讲解视频
   - 适用场景：
     - 产品讲解视频
     - 新闻播报
     - 虚拟主播
     - 教程讲解

### 阶段五：音图驱动视频（可选）

7. **口型同步视频**
   - 当需要静态图片"说话"时，调用 `audio-image-to-video` 技能
   - 用户提供静态图片（人物正面照）和音频文件
   - 自动生成口型与音频同步的视频
   - 适用场景：
     - 静态照片"说话"效果
     - 虚拟主播口播
     - 人物口型配音
     - 音频驱动的表情动画

## 参数配置

### 视频参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `ratio` | 宽高比 | `9:16` (竖版短视频) |
| `duration` | 单段视频时长 | `5` 秒 |
| `resolution` | 视频分辨率 | `720p` |
| `generate_audio` | 是否生成音频 | `true` |

### 图片参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `size` | 图片尺寸 | `2048x2048` |
| `style` | 风格提示词 | 自动推断 |

### 数字人参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `ref_video` | 参考视频（含数字人和语音） | 必填 |
| `text` | 数字人要说的文案 | 必填 |
| `audio_language` | 音频语言 | `zh` |

### 音图驱动视频参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `image` | 静态图片（人物正面照） | 必填 |
| `audio` | 音频文件（支持多个） | 必填 |
| `prompt` | 视频描述提示词 | 必填 |
| `model` | 模型名称 | `InfiniteTalk` |
| `motion_frame` | 运动帧数 | `9` |

## 技巧与最佳实践

### 提升视频质量

1. **Prompt 优化**
   - 使用具体、形象的描述
   - 包含场景、动作、情绪细节
   - 中英文混合使用效果更好

2. **分镜规划**
   - 每个分镜有明确的核心动作或内容
   - 分镜之间有逻辑连贯性
   - 控制分镜数量，短视频建议 3-6 个

3. **首尾帧设计**
   - 开头帧要有吸引力
   - 结尾帧要有收尾感或引导性

### 风格一致性

1. 在生成连续分镜时，使用相同的风格描述
2. 保持画面色调和构图风格统一
3. 必要时使用参考图来保持一致性

## 环境变量

确保以下环境变量已设置：

```bash
# 火山引擎 API Key（图片/视频生成）
export MODEL_VIDEO_API_KEY="your-video-api-key"
export MODEL_IMAGE_API_KEY="your-image-api-key"

# 或使用通用 Key
export MODEL_AGENT_API_KEY="your-general-api-key"

# Moark API Key（数字人视频生成）
export MOARK_API_KEY="your-moark-api-key"
```

## 注意事项

1. **API 限制**
   - 视频生成有并发限制，建议逐个生成
   - 视频 URL 有效期 24 小时，及时下载

2. **内容合规**
   - 确保生成内容符合平台规范
   - 避免侵权内容

3. **输出管理**
   - 所有图片和视频 URL **必须原样保留**，禁止任何修改
   - 建议生成后立即下载保存

## 故障排除

| 问题 | 解决方案 |
|------|----------|
| 视频生成失败 | 检查 API Key 配置，降低分辨率重试 |
| 风格不一致 | 使用参考图或增加风格描述 |
| 生成超时 | 减少分镜数量，缩短视频时长 |
